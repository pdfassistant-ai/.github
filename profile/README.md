## Native AI Integration via MCP

Your everyday AI tools are great at writing and analyzing but they hit a wall with heavy document processing. We bridge that gap. We offer a direct Model Context Protocol integration that gives your favorite AI real time access to our enterprise suite without ever leaving the chat interface. 

You can merge, split, compress, OCR, watermark, encrypt, and flatten forms dynamically using plain English prompts.

Read the full setup guide on our blog at **[pdfAssistant.ai Now Plugs Directly into Claude and ChatGPT](https://pdfassistant.ai/blog/announcements/pdf-assistant-now-plugs-directly-into-claude-chat-gpt-and-more/)** to see exactly how to connect.

**Frictionless Setup**
1. **Link Account:** Add the pdfAssistant.ai remote server endpoint `https://chat.pdfassistant.ai/mcp` to your AI client
2. **Secure Login:** Authenticate safely via an OAuth popup using your active Plus, Pro, or Enterprise account credentials
3. **Run Automations:** Drop a document into your chat window and describe your target to process the file in our secure cloud

**Supported Platforms**
* **Claude Desktop:** Run `claude mcp add --scope user --transport http pdfAssistant https://chat.pdfassistant.ai/mcp` in your terminal then launch Claude and type `/mcp`
* **ChatGPT Web:** Enable Developer Mode in your application settings and configure a remote app pointing to our official endpoint
* **Codex CLI:** Run `codex mcp add pdfAssistant --url https://chat.pdfassistant.ai/mcp` followed by `codex mcp login pdfAssistant`

Because our server is built to the open Model Context Protocol standard it is compatible with any AI platform that supports MCP over Streamable HTTP.

## Enterprise Security and Privacy

Trust is the foundation of our platform. We process your sensitive data with rigorous enterprise security standards and our controls are independently audited for [SOC 2 Type 2 compliance](https://trust.datalogics.com/).

* **Privacy First:** Your documents are never used to train our AI models or third party models
* **Secure Authentication:** Our integrations use standard OAuth 2.0 authentication so your credentials are never exposed or stored locally
* **Data Encryption:** All files are protected with TLS 1.3 in transit and AES 256 at rest
* **Ephemeral Storage:** Files are stored on isolated infrastructure and are automatically deleted within 24 hours
